# RB Packer: Context-Dump Generator

> Vereint: `packer.py` + `compare_projects.py`

## 1. Prerequisites

- [ ] Python 3.12+ verfügbar
- [ ] `scripts/packer.py` existiert im Skill-Ordner
- [ ] Mindestens ein Quellverzeichnis vorhanden (src/, backend/, etc.)

## 2. Instructions

### Standard-Modus (Auto-Detect)
1. **Ausführen:** `python .agent/skills/rb_packer/scripts/packer.py`
2. **Ergebnis:** Dump wird in `.rb_dumps/PROJECT_CONTEXT_DUMP_<timestamp>.txt` gespeichert
3. **Verifizieren:** Dateigröße prüfen (typisch: 10-500 KB)

### Custom-Modus (Verzeichnisse wählen)
```powershell
$env:RB_PACK_INCLUDE = "backend,frontend,docs"
python .agent/skills/rb_packer/scripts/packer.py
```

### Projekt-Vergleich
```powershell
python .agent/skills/rb_packer/scripts/compare_projects.py <pfad_a> <pfad_b>
```

## 3. Auto-Detection Logik

Der Packer scannt automatisch nach:

**Immer inkludiert (wenn vorhanden):**
- `docs/_rb/`, `src/`, `tests/`, `scripts/`

**Erkannt und inkludiert:**
- `backend/`, `frontend/`, `server/`, `client/`
- `app/`, `lib/`, `pkg/`, `packages/`
- `api/`, `web/`, `core/`

**Immer ausgeschlossen:**
- `.git`, `node_modules`, `.venv`, `__pycache__`
- `dist`, `build`, `.next`, `.nuxt`, `vendor`
- `.pytest_cache`, `coverage`, `.idea`, `.vscode`

**Geblockte Dateitypen:**
- `.env`, `.pem`, `.key`, `.db`, `.sqlite`, `.pyc`

## 4. Output-Format

```
# Project Context Dump: 2026-02-19_22-18
# Generated by RB-Framework Packer v2.0
# Repository: _rb-Protokoll
# Total files: 42

================================================================================
FILE: docs/_rb/02_SYSTEM_FACTS.md
================================================================================

[Dateiinhalt]
```

## 5. Constraints

- **Max Dateigröße:** 2 MB pro Datei (größere werden übersprungen)
- **Keine Secrets:** Geblockte Dateien werden nie in Dumps inkludiert
- **Dump-Verzeichnis:** Immer `.rb_dumps/` (wird automatisch erstellt)
- **Encoding:** UTF-8, fehlertoleranter Modus
