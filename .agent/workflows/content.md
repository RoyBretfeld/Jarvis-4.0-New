# /content – Content-Erstellung nach Protokoll

Dieser Workflow führt durch die strukturierte Erstellung von SEO- und KI-optimiertem Content.

## Schritte

1. **Skill-Instruktionen laden:**
   Lies `.agent/skills/content_protocol/SKILL.md` für den vollständigen Leitfaden.

2. **Phase bestimmen:**
   Frage: Wird neuer Content erstellt oder bestehender geprüft?
   - **Neu** → Starte bei Phase 1 (Recherche)
   - **Prüfung** → Starte bei Phase 3 (Technische Optimierung)

3. **Checkliste laden:**
   Lies `.agent/skills/content_protocol/templates/content_checklist.md` als Schnell-Referenz.

4. **Content erstellen/prüfen:**
   Arbeite die Phasen der Reihe nach ab:
   - Phase 1: Recherche & Keyword-Matrix
   - Phase 2: Content-Struktur (Hero → Kontext → Hauptteil → FAQ)
   - Phase 3: Schema.org + On-Page SEO + KI-Optimierung
   - Phase 4: Pre-Publish Checks
   - Phase 5: Post-Publish Monitoring

5. **Ergebnis melden:**
   ```
   📝 Content Protocol Report
   ═══════════════════════════
   Thema:        [Titel]
   Typ:          [Article|HowTo|FAQ]
   Keywords:     Primary: [X], Secondary: [N]
   Checkliste:   [X]/[Y] Punkte erfüllt
   Schema:       ✅ / ❌
   KI-optimiert: ✅ / ❌
   ═══════════════════════════
   Status:       BEREIT / NACHARBEIT NÖTIG
   ```
